### @method('PUT')を記述した行に何が出力されているか

### findメソッドの引数に指定しているIDは何のIDか
  - bladeファイルでrouteを呼び出したにrouteにリクエストとしてパラメータを受け取る(第一引数{id})
    routeからcontrollerメソッドに引き渡し
    blade -> route -> controllerの流れ
  - bladeファイルでDBからTodosテーブルのデータを取得、順番に表示している
    表示しているデータそれぞれが持っているIDを引き渡している

### findメソッドで実行しているSQLは何か
  - SELECT * FROM Todos WHERE ID = $id;

### findメソッドで取得できる値は何か
  - $id で指定したレコードを格納した連想配列
    key = カラム名 value = フィールド

### saveメソッドは何を基準にINSERTとUPDATEを切り替えているのか
  - Modelから直接fill()->save() を実行した場合はINSERT
    Modelからfind()で取得したオブジェクトからsave()を実行した場合はUPDATE
    Modelクラスは内部に$existsプロパティを所持(フラグ管理)
    インスタンスを生成された時にはfalseとなっている
    find()が呼び出されたときにTrueに切り替わり、trueに切り替わり、
    すでにあるデータと判定、UPDATEが実行される

### TodoControllerクラスのコンストラクタはどのタイミングで実行されるか

### RequestクラスからFormRequestクラスに変更した理由
  - FormRequest自体Requestクラスを継承しているためRequestクラスを拡張したもの
    リクエスト送信時のバリデーションや文字数制限などを管理
    - バリデーション
      入力された内容が適切か判断
      中身が空でないか、不適切な表現が含まれていないかなどなど

### $errorsのhasメソッドの引数・返り値は何か
  - $errors = bladeファイル内で使用できるメソッド
  - $errors -> has()
    - バリデーションエラーが発生しているかどうかをチェック
    - 引数 ・・・バリデーションチェックする文字列
    - 返り値 ・・・ boolean。True = エラーあり。False = エラーなし

### $errorsのfirstメソッドの引数・返り値は何か
  - $errors->first()
    複数のエラー内容があった際に最初の1つだけを返す
    例えばcontentsが文字数制限を超している、かつ不適切な内容が含まれていると２種類のエラー内容があった場合、
    最初の一つ、今回だと文字数制限を超していることだけを返す
  - 引数 ・・・ バリデーションチェックする文字列
  - 返り値 ・・・ FormRequestを継承したTodoRequest -> massages()で設定したエラーメッセージ
                エラーがなければ""を返す

### ORMとは何か、またLaravelが使用しているORMは何か
  - ORM
    クラスとDBを紐付けて(マッピング)することで
    SQLを直接操作せずにクラスからDBの操作が可能
    クラスのみで完結させることができるためsqlの知識が疎くても使用できる

  - Laravelが使用しているORM = Eloquent

### composer.json, composer.lockとは何か
  - composer
    phpのライブラリやパッケージを管理するツール
    ライブラリによっては他ライブラリが必要だったりと依存関係があるものを
    一度にインストールが可能

  - composer.json
    composer installコマンドでプロジェクトに必要なライブラリやパッケージをインストール際に参照するファイル
    これに書かれている情報をもとにダウンロード、インストールが行われる

  - composer.lock
    composer.jsonでダウンロード、インストールされたライブラリのバージョンやパッケージが書き込まれる
    jsonではバージョンなどの整合性を判断しながらインストールが行われるため時間がかかるが、
    すでにバージョンなどが記載されているためjsonより早くインストールが行われる
    composer.jsonでライブラリなどがインストールされた時に自動的に生成され、jsonより優先して呼ばれる
    
  - 要は最初に環境構築を行う時に.jsonが読み込まれて構築が行われる
    その後は.lockを他開発メンバーに渡せば早くインストールができる


### composerでインストールしたパッケージ（ライブラリ）はどのディレクトリに格納されるのか
  - プロジェクト内のvendorディレクトリ
    一つにまとまっている為、プロジェクト単位でライブラリやパッケージの管理が可能
